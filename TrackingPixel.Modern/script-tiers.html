TIER 1

Basic IMG Tag

Complexity: 1/5 ‚Äî Copy-paste a single line. No JavaScript knowledge required.

<img src="https://smartpixl.com/12345/1_ClientName_SMART.GIF" style="display:none!important"/>

~90 characters

Installation:

  1. Copy the code above
  2. Paste anywhere in your HTML (before </body> recommended)
  3. Done!

Sec-CH-UA (Browser)

~20 data points from HTTP headers only

‚úÖ Pros

  • Simplest possible installation
  • Works even if JavaScript is disabled
  • No technical knowledge required
  • Can be added via CMS/WYSIWYG editors

‚ùå Cons

  • No client-side data (screen, timezone, etc.)
  • Referer may be stripped by some browsers
  • Limited fingerprinting capability
TIER 2

Basic Script with Page URLs

Complexity: 2/5 ‚Äî Single-line script tag. Adds page URL tracking.

<script>(function(){new Image().src="https://smartpixl.com/12345/1_ClientName_SMART.GIF?url="+encodeURIComponent(location.href)+"&ref="+encodeURIComponent(document.referrer)})()</script>

~210 characters

Installation:

  1. Copy the code above
  2. Paste before </body> tag
  3. Done!

Data Captured:

All Tier 1 headers
Page URL (reliable)
Page Referrer (reliable)

~22 data points ‚Äî Headers + guaranteed page URL

‚úÖ Pros

  • Still very simple (one line)
  • Guarantees accurate page URL capture
  • Referrer explicitly passed (not browser-dependent)

‚ùå Cons

  • Requires JavaScript enabled
  • Still missing device/screen data
TIER 3

Enhanced Script with Device Data

Complexity: 3/5 ‚Äî Multi-line script. Captures screen, timezone, and device info.

<script>
(function(){
  var s=screen,n=navigator,d=new Date();
  new Image().src="https://smartpixl.com/12345/1_ClientName_SMART.GIF?"
    +"sw="+s.width+"&sh="+s.height+"&cd="+s.colorDepth
    +"&tz="+encodeURIComponent(Intl.DateTimeFormat().resolvedOptions().timeZone)
    +"&tzo="+d.getTimezoneOffset()+"&lang="+n.language
    +"&url="+encodeURIComponent(location.href)
    +"&ref="+encodeURIComponent(document.referrer);
})();
</script>

~450 characters

Installation:

  1. Copy the code above
  2. Paste before </body> tag
  3. Done!

Data Captured:

All Tier 1 & 2 data
Screen Width
Screen Height
Color Depth
Timezone (e.g., America/Chicago)
Timezone Offset
Browser Language

~28 data points ‚Äî Excellent for user identification

‚úÖ Pros

  • Screen + timezone = strong fingerprint
  • Helps distinguish users on same IP
  • Readable code (easy to audit)

‚ùå Cons

  • Multi-line (slightly harder to paste)
  • Some clients may want to review code
TIER 4

Full Device Fingerprint

Complexity: 4/5 ‚Äî Complete device fingerprinting. Maximum data collection.

<script>
(function(){
  var s=screen,n=navigator,d=new Date(),w=window;
  new Image().src="https://smartpixl.com/12345/1_ClientName_SMART.GIF?"
    +"sw="+s.width+"&sh="+s.height+"&cd="+s.colorDepth
    +"&pd="+(w.devicePixelRatio||1)
    +"&tz="+encodeURIComponent(Intl.DateTimeFormat().resolvedOptions().timeZone)
    +"&tzo="+d.getTimezoneOffset()
    +"&lang="+n.language+"&plt="+(n.platform||"")
    +"&cores="+(n.hardwareConcurrency||"")
    +"&mem="+(n.deviceMemory||"")
    +"&touch="+("ontouchstart"in w?1:0)
    +"&url="+encodeURIComponent(location.href)
    +"&ref="+encodeURIComponent(document.referrer);
})();
</script>

~620 characters

Data Captured:

All Tier 1-3 data
Pixel Ratio (Retina detection)
Platform (Win32, MacIntel, etc.)
CPU Cores
Device Memory (GB)
Touch Support

~33 data points ‚Äî Best fingerprint without canvas/WebGL

‚úÖ Pros

  • Highest accuracy user identification
  • CPU cores + memory = unique device signature
  • Detects mobile vs desktop reliably
  • No privacy warnings triggered

‚ùå Cons

  • Longer code to paste
  • deviceMemory is Chrome-only

<script src="https://smartpixl.com/js/12345/1.js" async></script>

~70 characters (but loads external file)

Data Captured:

All Tier 1-4 data

Installed Fonts Count
Audio Context Fingerprint
Scroll Depth (optional)

100+ data points ‚Äî Maximum possible collection

‚úÖ Pros

  • Simplest install (one short line)
  • Can update collection without client changes
  • Enables advanced fingerprinting
  • Can add scroll/time tracking

‚ùå Cons

  • Requires client to trust external JS
  • Some security teams will reject
  • Additional HTTP request
  • May trigger ad blockers

üéØ Recommendations

For Most Clients: Tier 3 or Tier 4

Tier 3 is the sweet spot ‚Äî meaningful data increase with readable code that clients can audit. The screen + timezone combination alone dramatically improves user identification.

Tier 4 if they're comfortable with slightly longer code. The CPU cores + device memory fingerprint is very powerful.

For Non-Technical Clients: Tier 1 or Tier 2

If the client just wants "paste this one line and forget it," go with Tier 2. It's still one line but ensures we get the page URL reliably.

For Enterprise/Tech-Savvy Clients: Tier 5

Only recommend if they're comfortable loading external JavaScript. This enables future enhancements without requiring them to update their code.
